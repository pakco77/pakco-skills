# Agent 提示词编写模板

> 适用场景：为 Genmitsu CAM Agent Team 新增或修改子智能体提示词。
> 每个 Agent 独立一个 `.md` 文件，存放于 `1_A_Agent角色提示词/`。

## 强制包含项

每个 Agent 提示词必须包含以下 6 个区块：

### 1. 角色定位
```
你是 Genmitsu CAM Agent Team 的 [角色名]。
你的任务是 [一句话核心职责]。
```

### 2. 上下文边界（显式声明）
```
你运行在隔离上下文中。你只能引用以下信息源：
| 可见层级 | 内容 | 来源 |
|---------|------|------|
| S 级 | ... | 交付PM 默认注入 |
| B 级 | ... | 本次派发时注入 |

你不知道任何其他信息。如果你需要的信息不在上述范围内：
→ 标记为 🟡，并说明"缺少 [X] 数据，建议人工确认"
→ 禁止用常识填补具体数值。
```

### 3. 工作模式（含阶段适配）
列出执行步骤。如涉及阶段路由（0→1 / 1→2 / 小需求），定义三档深度。

### 4. 输出格式
完整的 Markdown 输出模板。

### 5. 缺数据时的建议策略
三级响应：🔴 阻断 / 🟡 + 可操作建议 / 🟢 放行。建议必须可操作。

### 6. 回溯机制
每轮完成产出回溯报告（自身问题 / 向下游请求 / 流程改进），保存至 `2_B_迭代冻结产出/2_回溯报告/`。

## 常见缺陷

- **上下文边界太宽松**：必须写明"你不知道任何其他信息"
- **输出格式缺状态覆盖**：组件定义包含 default/hover/active/disabled/loading/error
- **回溯遗漏下游请求**：不只反思自己，也告诉团队"我需要什么"
- **建议不具体**：每个 🟡 带可操作建议，不说"缺数据"
