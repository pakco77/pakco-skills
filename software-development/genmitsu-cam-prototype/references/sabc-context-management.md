# SABC 上下文管理系统

> Genmitsu CAM Agent Team 的上下文注入规则。

## 核心矛盾

上下文窗口有限 vs 任务复杂度无限。解法不是更大的窗口，而是裁剪注入。

## 信息暴露层次

| 标签 | 内容 | 注入规则 | 谁可见 |
|------|------|----------|--------|
| **S** | 系统级：铁律、红绿灯规则、SABC本身、硬件参数库、市场调研、产品方针、专业名词库 | 全链路永久注入 | 所有人 |
| **A** | Agent 专题提示词 | 默认注入对应 Agent | 所有人 |
| **B** | 上游已冻结的迭代产出 | 交付PM 摘取相关片段注入下游 | 跨 Agent |
| **C** | 预留 / 冗余 | — | — |

Agent 工作草稿内部浮动，不对外暴露。

## 依赖失效标记 [STALE]

B 级文档被修改时（如跨职能回环修复），所有依赖它的下游产出标 `[STALE]`。

```
UX骨架 B → 被修改 → UI规范 [STALE], prototype.html [STALE], qa-report.md [STALE]
```

交付PM 派发时跳过 [STALE] 产出，上游更新后重跑。闭环完成后清除。

## 上下文摘要注入

交付PM 不对下游丢整份文档，而是摘取相关片段。

| 下游 Agent | 从 PRD 摘取 | 从其他摘取 |
|-----------|------------|-----------|
| UI Agent | 组件树 + 交互定义 | UX 骨架 |
| 前端 Agent | 验收标准 | UI 设计规范 |
| 测试 Agent | 验收标准 | UI 组件状态表 |

## 变更差异通知

B 级文档更新后，交付PM 生成 diff 摘要给下游。回环修复时前端只改 UX 更新的点。

## 日志规范

文件：`[feature]-trace.log`
格式：`[时间戳] [节点] [动作] [结果] [版本]`
动作类型：`派发` `完成` `裁决` `STALE` `回环` `异常`

**不记录 Agent 思考过程**，只记录决策和状态变更。

## 上下文注入模板

交付PM 派发子代理时使用：

```
S级规则（永久注入）：
- 三条铁律 / 红绿灯约束

A级规则（本 Agent 专属）：
- [该 Agent 的提示词全文]

B级上下文（上游产出摘要）：
- [摘录的相关片段]

任务：
- [具体交付物和约束]

你必须只引用已注入的上下文，不假设未注入信息。
完成后返回文件路径，不返回全文。
```
