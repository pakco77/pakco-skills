# 公众号后台数据解析

## 获取数据

1. 公众号后台 → 内容分析 → 右上角「导出CSV」
2. 得到 tendency_*.xls 文件（实际是 Excel，非 CSV）
3. 放入 `写作Agent/数据/`

## XLS 结构（tendency 文件）

文件有 3 个数据区域，分布在 16 列中：

### 左侧（列 1-3）：每日阅读渠道
- 日期、渠道（公众号消息/聊天会话/朋友圈/公众号主页/推荐/搜一搜/其他/全部）、阅读量
- "全部"行 = 当日总阅读

### 中间（列 5-9）：每日互动
- 日期、分享人数、原文阅读人数、收藏人数、发表篇数

### 右侧（列 11-14）：文章汇总
- 传播渠道（全部/推荐）、发表日期（如 20260413）、内容标题、阅读量
- 筛选"全部"渠道即得每篇文章总阅读量

## 关键数据洞察

- **阅读来源分布**：推荐通常占 85-90%（算法号特征）
- **朋友圈占比**：<5% 说明转发不足
- **头部效应**：TOP1 通常占总阅读 50%+
- **脉冲模式**：爆款周 → 下一周打回原形 → 缺乏流量承接

## 脚本

```bash
python3 ~/Documents/Obsidian\ Vault/写作Agent/wechat_weekly.py
```

自动解析最新 tendency XLS，生成周报到 `输出/周报-YYYY-MM-DD.md`。
