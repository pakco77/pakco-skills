# Hybrid 模式大纲范例

> 来源：2026-05-10 用户亲自修订的大纲
> 标题：「我是办公室第一个用HTML汇报的人，一个月后入职了新公司」
> 核心论点：AI时代，MD是给AI看的，HTML是给人看的
> 字数：3500-4000

## 结构拆解

### 钩子（反差自曝型）
"我先坦白一件事。我自学Python失败过三次，至今分不清`<div>`和`<span>`。但上个月我做的HTML分析报告，被我前老板收藏到了浏览器书签栏。"
→ 公式：短板 × 意外成果 = 钩子张力

### 第一段：那个不想做Word的下午（Pakco 场景叙事）
- 产品经理身份，经常做竞品分析、功能调研
- 每次交word/excel，老板都回"太长了，总结一下"
- 打开VS Code，让Agent写HTML（不是自己写代码）
- 真实动机：改内容不用调格式
- 老板反馈："这是你自己做的？""以后就这样。"

### 第二段：从裸搓到发现huashu-md-html（Kazel 叙事深描）
- 一开始无skill，让AI直接出HTML——能用但丑
- 直到发现 huashu-md-html
- 转折：不是"排版更好看"，而是"不用管排版"
- 写MD→工具输出HTML——像下载了一个App

### 第三段：工作从此只分两种（论点展开）
- 对AI用MD交接——结构化、纯文本、AI秒懂
- 对同事用HTML交接——可视化、可交互、人眼三秒定生死
- MD是协议层，HTML是界面层
- 核心论点：AI时代，MD是给AI看的，HTML是给人看的

### 第四段：想去更"AI"的团队（Kazel 价值反转）
- 不是"HTML写得好所以跳槽"
- 反转：在AI时代，最重要的技能不是做东西，是定义东西
- 不需要会写HTML，需要知道"该输出什么HTML"
- 看懂了人机协作的分界线

### 升华段：你也可以
- 无代码基础，Python三战三败
- 但产品级HTML，只需一个Agent + 一个skill
- 那个下午打开VS Code而不是Word——本质是回答了：未来工作，哪些交给AI，哪些留给人

### 结尾（Pakco 固定结尾）
"谢谢你读到这里，无声感激。我是Pakco，点个关注，共同进步。"

## 关键学习

1. **用户提供钩子素材时，直接用，不要替换**——Skill模板是"用户没给素材时"的备选
2. **Hybrid模式的分工**：骨架（开场+结尾+论点框架）用Pakco，故事段落（转折+深描+反转）用Kazel
3. **核心论点由用户提炼**：用户说的"MD是给AI看的，HTML是给人看的"比AI生成的论点更锋利，直接作为文章中枢
